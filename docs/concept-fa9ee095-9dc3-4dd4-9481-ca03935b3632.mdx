# \[WS\] Verify Elements Count

## Description

Verify the number of expected elements \(JSON/XML\) in the response \(output\) of a web service call.

Keyword name: <code>WS.verifyElementsCount</code>

## Parameters

<table><tr><th>Param</th><th>Param Type</th><th>Mandatory</th><th>Description</th></tr><tbody><tr><td>response</td><td>ResponseObject</td><td>Required</td><td>The object that represents an HTTP response.</td></tr><tr><td>locator</td><td>String</td><td>Required</td><td>An expression Katalon will use to go through and look for the expected element(s). You can refer to this guide: <a href="/create-tests/test-objects/api-test-objects/handle-response-messages-in-katalon-studio" >Handle Web Service Response object</a>. The syntax for locator is GPath. To learn more about GPath, see this <a href="http://groovy-lang.org/processing-xml.html" >guide</a>.</td></tr><tr><td>count</td><td>int</td><td>Required</td><td>The expected number of element(s) should appear in the responded data (usually JSON/XML)</td></tr><tr><td>flowControl</td><td>FailureHandling</td><td>Optional</td><td>Specify <a href="/maintain/configure-failure-handling-settings-in-katalon-studio" >failure handling</a> schema to determine whether the execution should be allowed to continue or stop.</td></tr></tbody></table>

## Returns

-   <code>true</code>, if your expectation is met, otherwise <code>false</code>.
