# Value types in Katalon Studio

While designing automation tests in Katalon Studio, you might need to configure data for certain test steps, such as:

-   Pass input parameters to a test step.
-   Get the output value from a test step.

You can select from a wide range of value types supported by Katalon Studio as below:

<table><tr><th>Value Type</th><th>Description</th></tr><tbody><tr><td>Constant</td><td>The data is a value that will not change over time.</td></tr><tr><td>Variable</td><td>The data is a value that might be changed during execution. This type of data can only be referred to within the scope of the test case where it is defined.</td></tr><tr><td>Global Variable</td><td>The data is a value that might be changed during execution. This type of data can be referred to anywhere in the project.</td></tr><tr><td>Test Data Value</td><td>The data is a value coming from a test data file.</td></tr><tr><td>Binary</td><td>The data is a binary expression.</td></tr><tr><td>Boolean</td><td>The data is a Boolean value that can be evaluated from a Boolean expression.</td></tr><tr><td>Test Case</td><td>The data is a test case.</td></tr><tr><td>Test Data</td><td>The data is a test data file.</td></tr><tr><td>Test Object</td><td>The data is a test object.</td></tr><tr><td>Method Call</td><td>The data is a method call.</td></tr><tr><td>Property</td><td>The data is a public property value of an object.</td></tr><tr><td>List</td><td>The data is a list of values.</td></tr><tr><td>Dictionary</td><td>The data is a collection of keys and values.</td></tr><tr><td>Number</td><td>This is a numeric value of which data type can be short, long, integer, double, float, byte.</td></tr></tbody></table>
