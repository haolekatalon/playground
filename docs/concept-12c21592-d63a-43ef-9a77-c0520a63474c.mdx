## Jira integration in Katalon Platform

This section shows you how to integrate Jira with Katalon Platform.

### Jira integration in Katalon TestOps

To enable Katalon TestOps - Jira integration, follow this guide: [Enable Jira integration in Katalon Studio](/organize/integration-for-organizing-tests/jira-integration/configure-jira-integration-in-katalon-studio#).

Katalon TestOps - Jira integration allows you to:

-   Link test cases to Jira requirements. See [Link test cases to Jira requirements](/plan/integration-for-test-planning/link-test-cases-to-jira-requirements).
-   Link test runs to a populated Jira release. See [Link test runs to a release in TestOps](/plan/link-test-runs-to-a-release-in-testops).
-   Link test runs to Jira defects. See [Link test runs to Jira defects in Katalon TestOps](/analyze/integration-for-test-analyzing/jira-integration/link-test-runs-to-jira-defects-in-katalon-testops).
-   View test results in Jira Test Results gadget. See [View test results in the Jira Test Results gadget](/analyze/integration-for-test-analyzing/jira-integration/view-test-results-in-the-jira-test-results-gadget).

### BDD testing with Jira integration

If you want to conduct BDD testing and manage them in Jira, refer to [Manage BDD test cases with Jira integration](/organize/integration-for-organizing-tests/jira-integration/manage-bdd-test-cases-with-jira-integration).

Next, to import BDD test cases for execution in Katalon Studio, you need to configure Jira integration in Katalon Studio. See [Enable Jira integration in Katalon Studio](/organize/integration-for-organizing-tests/jira-integration/configure-jira-integration-in-katalon-studio#).

Katalon Studio provides a seamless Jira integration that allows you to:

-   Write BDD scenarios in Jira tickets.
-   Import BDD test cases from Jira to Katalon Studio.
-   Link test execution results with the corresponding Jira tickets.

You can also view the test results in Katalon TestOps. See [View BDD test results in Katalon TestOps](/analyze/reports/view-test-reports/view-test-reports-in-katalon-testops/view-bdd-test-results-in-testops).

## BDD Testing Framework \(Cucumber integration\)


:::note

-   For better performance, you should clean up the Katalon workspace frequently. Navigate to **File** &gt; **Clean up**.


:::

Behavior-driven development \(BDD\) testing framework allows you to create test cases in plain English. BDD testing framework focuses on the behavior of the product and user acceptance criteria. Cucumber is a BDD framework tool to write test cases, which uses Gherkin, an ordinary language parser, that permits writing scripts in English.

The integration of the Cucumber framework in Katalon Studio allows you to include Cucumber test hooks, which work at the start and the end of a scenario in a behavior-driven development \(BDD\) test. To learn more about test hooks in the Cucumber framework, you can refer to Cucumber document: [Cucumber Hooks reference](https://cucumber.io/docs/cucumber/api/#hooks).

With the BDD testing framework in Katalon Studio, you can:

-   Create or add a feature file.

-   Define and link steps.

-   Set the default package for step definitions.

-   Run your feature file.

-   Add your feature file to a test case.

-   View BDD report files generated from Katalon Studio.

-   Upload and view BDD reports on Katalon TestOps.


To start working with BDD feature files and use test hooks for your BDD framework, see [Working with BDD feature files in Katalon Studio](/create-tests/manage-test-artifacts/working-with-bdd-feature-files-in-katalon-studio).

To learn how to create BDD test cases, see [Create BDD test case in Katalon Studio](/create-tests/create-test-cases/create-bdd-test-cases-in-katalon-studio).

You can download the sample project here on our GitHub repository: [Katalon BDD Cucumber Tests](https://github.com/katalon-studio-samples/katalon-bdd-cucumber-tests).
